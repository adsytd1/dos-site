Задача

KazGeoTech — нефтесервисная компания. HR-отдел тратил часы на первичный скрининг кандидатов: звонки, анкеты, базовые вопросы. При этом подходящих кандидатов часто теряли из-за долгого отклика.

Нужен был бот, который принимает заявки кандидатов, задаёт квалификационные вопросы, ведёт базу данных и уведомляет HR-команду о подходящих кандидатах.

Решение

Мы построили полноценную HR-автоматизацию на базе n8n — платформы для workflow-автоматизации. Система принимает обращения кандидатов через WhatsApp, квалифицирует их с помощью ИИ и сохраняет всю информацию в базу данных PostgreSQL.

  • n8n workflow — автоматическая обработка входящих сообщений через WhatsApp webhook
  • ИИ-рекрутер на базе GPT для квалификации кандидатов по профилю вакансии
  • PostgreSQL — структурированная база данных всех кандидатов и диалогов
  • Обработка мультимедиа: текст, голосовые сообщения, документы, фото
  • Система уведомлений для HR-команды о подходящих кандидатах
  • Управление ИИ через кнопку on/off для каждого диалога

Как работает HR-бот

Система работает через WhatsApp webhook и автоматически обрабатывает каждое входящее сообщение:

1. Приём сообщения. Кандидат пишет на WhatsApp номер компании. Webhook принимает сообщение и определяет его тип: текст, аудио (голосовое сообщение), документ (резюме, сертификаты) или изображение.

2. Маршрутизация по типу контента. Каждый тип сообщения обрабатывается по-разному. Текстовые сообщения идут напрямую в ИИ. Голосовые сообщения сначала транскрибируются в текст. Документы и изображения сохраняются в базу данных с привязкой к профилю кандидата.

3. ИИ-квалификация. Бот задаёт кандидату вопросы по профилю вакансии: опыт работы в нефтесервисе, наличие сертификатов, готовность к командировкам, владение специализированным оборудованием. ИИ анализирует ответы и формирует оценку соответствия вакансии.

4. Сохранение в базу данных. Вся информация о кандидате сохраняется в базу данных: контакт, история диалога, прикреплённые документы, оценка ИИ, статус кандидата. HR-команда получает структурированное досье.

5. Уведомление HR-команды. Когда ИИ определяет, что кандидат соответствует требованиям — HR-менеджер получает уведомление с полным саммари: кто кандидат, какой опыт, какие документы приложил и рекомендация бота.

6. Ручной контроль. HR-команда может включать и выключать ИИ для конкретного диалога. Если кандидат проходит первичную квалификацию и нужен живой разговор — HR переключает бота в ручной режим и продолжает общение сам.

Техническая архитектура

  • n8n — платформа автоматизации, оркестрирует весь workflow
  • WhatsApp Business API — приём и отправка сообщений через webhook
  • PostgreSQL — база данных для хранения диалогов, документов и оценок
  • GPT API — квалификация кандидатов по профилю вакансии
  • Whisper API — транскрибация голосовых сообщений в текст
  • Маршрутизатор типов контента — обработка text/audio/document/image
  • Фильтрация групповых чатов — бот реагирует только на личные сообщения
  • AI toggle per conversation — кнопка on/off для управления ИИ
